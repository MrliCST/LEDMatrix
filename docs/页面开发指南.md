## 一、include/core/ — 核心公共模块

### 1.1 config.h — 引脚 & 常量 & 枚举

所有页面的 `#include "core/config.h"`（通过 `page_base.h` 间接引入）。

**引脚定义**（来自 `diagram.json`，不要改）：

```cpp
LED_DATA_PIN   23   // LED 矩阵 DIN
AUDIO_IN_PIN   16   // 麦克风
BTN1_PIN       21   // 按键1 (红)
BTN2_PIN       19   // 按键2 (绿)
BTN3_PIN        5   // 按键3 (蓝)
BUZZER_PIN     35   // 蜂鸣器
RANDOM_SEED_PIN 34  // 悬空，randomSeed 用
```

**你的页面需要关心的常量**：

| 宏 | 值 | 用途 |
|---|---|---|
| `MATRIX_WIDTH` | 32 | 屏幕宽度（像素） |
| `MATRIX_HEIGHT` | 8 | 屏幕高度（像素） |
| `BRIGHTNESS_MIN` | 5 | 亮度下限 |
| `BRIGHTNESS_MAX` | 145 | 亮度上限 |
| `BRIGHTNESS_SPACING` | 5 | 亮度步进 |
| `ANIM_INTERVAL1` | 60 | 动画1 帧间隔 (ms) |
| `ANIM_INTERVAL2` | 80 | 动画2 帧间隔 (ms) |
| `ANIM_INTERVAL3` | 100 | 动画3 帧间隔 (ms) |

**你的页面需要用到的枚举**：

```cpp

```

### 1.2 store.h — 全局数据仓库（只此一处）

**原则**：所有持久化数据都在 `Store` 里，页面**不自行创建全局变量**存配置。

```cpp
Store::instance()   // 获取单例引用
```

#### 5 个只读数据入口

```cpp
Store::instance().wifi()       // → const WifiConfig&   { ssid, pass, apConfig }
Store::instance().color()      // → const ColorConfig&  { r, g, b, mainColor, weekColor }
Store::instance().mode()       // → const ModeConfig&   { timeMode, rhythmMode, animMode }
Store::instance().clock()      // → const ClockConfig&  { hour, minute, bellIndex, enabled }
Store::instance().brightness() // → int                  亮度值
```

#### 只读引用 — 要改必须调 save

```cpp
// ✅ 正确：读数据
int cur = Store::instance().mode().timeMode;

// ✅ 正确：改数据 — 取副本 → 改 → 保存
ModeConfig cfg = Store::instance().mode();  // 拷贝一份
cfg.timeMode = TIME_FORMAT_DATE;
Store::instance().saveMode(cfg);            // 写回 NVS

// ❌ 错误：返回的是 const 引用，编译报错
Store::instance().mode().timeMode = 3;
```

#### 5 个写入方法

```cpp
Store::instance().saveWifi(cfg);       // WiFi 配置
Store::instance().saveColor(cfg);      // RGB 颜色
Store::instance().saveMode(cfg);       // 页面子模式
Store::instance().saveClock(cfg);      // 闹钟配置
Store::instance().saveBrightness(val); // 亮度
```

#### Store 里的数据结构一览

```
Store::instance()
├── wifi()   → WifiConfig    { ssid, pass, apConfig }
├── color()  → ColorConfig   { r, g, b, mainColor, weekColor }
├── mode()   → ModeConfig    { timeMode, rhythmMode, animMode }
├── clock()  → ClockConfig   { hour, minute, bellIndex, enabled }
└── brightness() → int
```

**记住**：页面需要存状态 → 存成自己的 `private` 成员变量（如 `_barHeights[32]`）。需要持久化的配置 → 走 `Store`。不要自己 `Preferences.begin()`。

### 1.3 state_machine.h — 状态机

#### 它是什么

```
┌──────────────────────────────────────────────────────┐
│                    StateMachine                       │
│  _pages[6] = { Setting, Time, Rhythm, Anim,          │
│                Clock, Bright }                       │
│  _current = Page::TIME    ← 当前是哪个页面            │
│                                                      │
│  update()  ───→  _pages[_current]->update()          │
│  onButton() ──→  _pages[_current]->onBtnXXxx()       │
│  gotoPage() ──→  旧页面 exit() → 新页面 enter()       │
└──────────────────────────────────────────────────────┘
```

#### 运行时流程

```
setup()
  │
  ├─ StateMachine::init()    创建 6 个页面对象，默认 _current = TIME
  │
loop() 每 10ms 跑一次:
  │
  ├─ Buttons::tick()        检测按键 → 触发 onButton() → 当前页面的按键方法
  ├─ StateMachine::update()  调用 _pages[_current]->update()
  └─ delay(10)
```

#### 页面切换流程

```
用户按下按键3 → Buttons 检测 → StateMachine::onButton(3, SHORT_PRESS)
  → _pages[TIME]->onBtn3Short()    // 你的 TimePage 代码
  → StateMachine::gotoPage(RHYTHM) // 你要跳转

gotoPage(RHYTHM) 内部:
  Step 1: _pages[TIME]->exit()      // 旧页面收尾
  Step 2: _current = RHYTHM          // 切换
  Step 3: _pages[RHYTHM]->enter(TIME) // 新页面初始化（知道从哪来的）
```

#### 你的页面只需要调这一个方法做跳转

```cpp
StateMachine::instance().gotoPage(Page::RHYTHM);  // 跳去节奏灯
```

## 二、include/pages/page_base.h — 页面基类

```cpp
class PageBase {
public:
    virtual void enter(Page from) {}   // 进入本页面时调用，from = 从哪个页面跳来的
    virtual void exit() {}             // 离开本页面时调用
    virtual void update() = 0;         // ⚠️ 必须实现！每帧调用

    virtual void onBtn1Short() {}      // 按键1 短按
    virtual void onBtn2Short() {}      // 按键2 短按
    virtual void onBtn3Short() {}      // 按键3 短按
    virtual void onBtn1Long()  {}      // 按键1 长按
    virtual void onBtn2Long()  {}      // 按键2 长按
    virtual void onBtn3Long()  {}      // 按键3 长按
};
```

| 方法 | 调用时机 | 是否必须覆写 |
|---|---|---|
| `enter(from)` | 状态机跳转到本页面时 | 推荐覆写，用于初始化状态 |
| `exit()` | 状态机离开本页面时 | 可选 |
| `update()` | 每 10ms 被 `loop()` 调用 | **必须**（`= 0`） |
| `onBtn1~3Short()` | 对应按键短按时 | 按需覆写 |
| `onBtn1~3Long()` | 对应按键长按时 | 按需覆写 |

**不覆写的方法 = 空实现**，按了没反应。只覆写你需要的。

## 三、你的代码是怎么被调用的 — 完整链路

以用户在 TIME 页面短按按键3 为例：

```
物理按键3 按下 0.5 秒松开
        │
        ▼
OneButton 库检测到短按
        │
        ▼
Buttons::_onBtn3Click()
  → StateMachine::instance().onButton(3, ButtonEvent::SHORT_PRESS)
        │
        ▼
StateMachine::onButton(3, SHORT_PRESS)
  → auto* pg = _pages[(int)_current];     // 拿到 TimePage
  → pg->onBtn3Short();                    // 调用你的方法
        │
        ▼
TimePage::onBtn3Short()     ← 你写的代码在这里执行！
  → StateMachine::instance().gotoPage(Page::RHYTHM);
        │
        ▼
  Step 1: TimePage::exit()                // 旧页面收尾
  Step 2: _current = Page::RHYTHM
  Step 3: RhythmPage::enter(Page::TIME)   // 新页面知道是从 TIME 来的
```

## 四、怎么写一个页面 — 模板

```cpp
// ===== include/pages/xxx_page.h =====
#ifndef XXX_PAGE_H
#define XXX_PAGE_H
#include "pages/page_base.h"

class XxxPage : public PageBase {
public:
    void enter(Page from) override;     // 初始化状态
    void update() override;             // 每帧逻辑
    void onBtn1Short() override;        // 需要的按键
    void onBtn3Short() override;        // 跳转到下一页面

private:
    // 你自己的临时状态 — 不需要持久化的放这里
    int _myState = 0;
    void _drawSomething();              // 绘制辅助方法
};

#endif
```

```cpp
// ===== src/pages/xxx_page.cpp =====
#include "pages/xxx_page.h"
#include "core/store.h"           // 读配置
#include "core/state_machine.h"   // 页面跳转
#include "hal/led_matrix.h"       // 画屏

void XxxPage::enter(Page from) {
    LedMatrix::instance().clear();
}

void XxxPage::update() {
    // 每 10ms 调用一次
    LedMatrix::instance().clear();

    // ... 你的绘制逻辑 ...
    // LedMatrix::instance().drawPixel(x, y, color);
    // LedMatrix::instance().setCursor(0, 0);
    // LedMatrix::instance().print("hello");

    LedMatrix::instance().show();
}

void XxxPage::onBtn1Short() {
    // 读配置
    int cur = Store::instance().mode().xxxMode;
    // 改配置
    ModeConfig cfg = Store::instance().mode();
    cfg.xxxMode = cur + 1;
    Store::instance().saveMode(cfg);
}

void XxxPage::onBtn3Short() {
    StateMachine::instance().gotoPage(Page::NEXT_PAGE);
}
```

## 六、禁止事项

| ❌ 不要这样 | ✅ 应该这样 |
|---|---|
| 自己 `Preferences prefs; prefs.begin()` | 用 `Store::instance().saveXxx()` |
| 新建全局变量 `int g_myVar;` | 放类的 `private` 成员里 |
| 在 `update()` 里 `delay()` | 用 `millis()` 做非阻塞计时 |
| 直接操作 `matrix.raw()` | 走 `LedMatrix::instance()` 封装方法 |
| 在页面 .h 里存持久化数据 | 持久化数据走 `Store`，页面只存临时状态 |
| `new` / `malloc` 动态分配 | ESP32 内存有限，用固定大小的成员数组 |
