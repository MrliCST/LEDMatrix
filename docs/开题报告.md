## 一、项目概述

EasyMatrix 是一款基于 **ESP32 + 32×8 NeoPixel LED 矩阵** 的多功能桌面摆件，集成拾音器（麦克风）、蜂鸣器和三个物理按键，提供 **WiFi 配网、NTP 时钟、频谱可视化、动画、闹钟** 等 6 个功能页面。

| 项目 | 说明 |
|---|---|
| 芯片 | ESP32 DevKit-C V4 |
| 显示 | 4 × 8×8 NeoPixel 矩阵，组成 32×8 像素屏幕（GRB 色序） |
| 输入 | 3 × 按键（短按 / 长按） |
| 音频 | 模拟麦克风（ADC 采样 + FFT） + 无源蜂鸣器（PWM 播放） |
| 通信 | WiFi 802.11 b/g/n（STA + SoftAP），NTP 网络对时 |
| 框架 | Arduino（C++17），FreeRTOS 多任务 |
| 平台 | PlatformIO IDE + Wokwi 在线仿真 |
| 版本控制 | Git（Gitee），Fork + Pull Request 协作 |

### 产品原型 — 6 个功能页面

```
  SETTING          TIME           RHYTHM          ANIM           CLOCK          BRIGHT
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ WiFi配网  │   │ 14:30:25 │   │ ▂▄▆█▆▄▂▁ │   │ 数字雨滴  │   │ 闹钟 07:30 │   │ 亮度: 45 │
│ 手机连接  │   │ 时:分:秒  │   │ 彩虹频谱  │   │ 渐变擦拭  │   │ 铃声: 2   │   │ █████░░░░│
│ 192.168  │   │ 06/03 周一│   │ 峰值下落  │   │ 像素闪烁  │   │ [开]      │   │ [+] [-]  │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
```

| 页面 | 功能 | 子模式 |
|---|---|---|
| SETTING | WiFi 配网（AP 热点 + Web 页面） | — |
| TIME | NTP 网络时间显示 | 3 种格式：时:分:秒 / 时:分 / 日期 |
| RHYTHM | 麦克风拾音 → FFT → 频谱可视化 | 4 种：彩虹柱 / 渐变柱 / 垂直渐变 / 纯峰值 |
| ANIM | 离线装饰动画 | 3 种：数字雨 / 颜色擦拭 / 像素闪烁 |
| CLOCK | 闹钟设置 + 铃声播放 | 可调节时/分/铃声/开关 |
| BRIGHT | LED 亮度调节 | 步进 5，范围 5～145 |

页面跳转链路：`SETTING → TIME → RHYTHM → ANIM → CLOCK → BRIGHT → TIME`

## 四、软件设计

### 4.1 设计原则

| 原则 | 实现方式 |
|---|---|
| **单一职责** | 每个类只做一件事：`Store` 只管数据、`LedMatrix` 只管画屏、`Buzzer` 只管发声 |
| **依赖倒置** | 页面只依赖 `PageBase` 接口和 HAL 层封装，不直接碰硬件寄存器 |
| **开闭原则** | 新增页面只需新建一个 `PageBase` 子类并在 `StateMachine::init()` 注册 |
| **最少知识** | 页面只通过 `Store::instance()` 读写配置，不自己操作 NVS |

### 4.2 单例模式

所有模块通过 **函数内 static 局部变量** 实现线程安全的懒汉式单例：

```cpp
// 每个模块都这样写 — 全局只此一份实例
Store& Store::instance() {
    static Store s;    // C++11 保证只初始化一次
    return s;
}

// 使用时
Store::instance().mode().timeMode;    // 读配置
LedMatrix::instance().drawPixel(0, 0, color);  // 画屏
```

全局无 `extern` 变量，无 `new`/`delete`，外部无法直接创建对象（构造函数是 `private`）。

### 4.3 页面状态机

```
StateMachine 持有 6 个 PageBase* 和一个当前页枚举 _current

  _pages[0~5] = { SettingPage, TimePage, RhythmPage, AnimPage, ClockPage, BrightPage }
  _current = Page::TIME

核心方法：

  gotoPage(Page p):
    ① _pages[旧]->exit()       // 旧页面收尾
    ② _current = p              // 切换
    ③ _pages[新]->enter(旧)     // 新页面初始化

  onButton(idx, event):
    ① 拿到 _pages[_current]     // 当前页面
    ② 根据 event 类型调 onBtnXxx()  // 只调当前页，不涉及其他页

  update():
    ① 拿到 _pages[_current]
    ② 调其 update()
```

**按键逻辑不在状态机里**，状态机只做分发。每个页面对按键的响应在各自类的 `onBtn1Short()` 等方法中实现，天然解耦。

### 4.4 硬件抽象层 (HAL)

| 类 | 封装内容 | 对外接口 |
|---|---|---|
| `LedMatrix` | Adafruit_NeoMatrix 初始化、坐标映射、颜色转换 | `drawPixel()` `print()` `clear()` `show()` |
| `Microphone` | ADC 等间隔采样 1024 点、arduinoFFT、15 频段分桶 | `sample()` → `bands[32]` |
| `Buzzer` | LEDC PWM 通道配置、FreeRTOS 播放任务 | `playSong(idx)` `stop()` `isPlaying()` |
| `Buttons` | OneButton 消抖/计时、3 个按键中断回调 | `begin()` `tick()` → 自动推 StateMachine |

页面开发者不接触 `Adafruit_NeoMatrix`、`arduinoFFT`、`OneButton` 等第三方库的 API，只调 HAL 层封装。

### 4.5 全局数据仓库（Store）

所有需要持久化的配置（WiFi 密码、颜色、模式、闹钟、亮度）集中在一个地方：

```cpp
Store::instance()
├── wifi()       → WifiConfig    { ssid, pass, apConfig }
├── color()      → ColorConfig   { r, g, b, mainColor, weekColor }
├── mode()       → ModeConfig    { timeMode, rhythmMode, animMode }
├── clock()      → ClockConfig   { hour, minute, bellIndex, enabled }
└── brightness() → int
```

- **读**：返回 `const` 引用，外部不可修改
- **写**：取副本 → 修改 → 调 `saveXxx()`，自动写入 ESP32 的 NVS（非易失存储）
- 页面**不在自己的类里存持久化数据**，只存临时状态（如动画帧计数、柱高度缓冲）

## 六、团队分工

### 人员安排（5 人）

| 角色 | 姓名 | 职责 | 涉及文件 |
|---|---|---|---|
| **负责人** | (你) | 架构设计、核心模块、git 仓库管理、代码评审 | `core/*` `hal/*` `net/*` `main.cpp` `platformio.ini` |
| **组员 A** | | TIME 页面 + CLOCK 页面 | `time_page.cpp` `clock_page.cpp` |
| **组员 B** | | RHYTHM 页面 + 麦克风调试 | `rhythm_page.cpp` `microphone.cpp` |
| **组员 C** | | ANIM 页面 + 位图调色 | `anim_page.cpp` `img/elems/*` |
| **组员 D** | | SETTING 页面 + BRIGHT 页面 + 配网页 HTML | `setting_page.cpp` `bright_page.cpp` `data/index.html` |

### 分工依据

| 考量 | 说明 |
|---|---|
| **耦合度** | TIME 和 CLOCK 都依赖 `time.h` 和 NTP，分给同一人减少沟通成本 |
| **难度均衡** | RHYTHM 涉及 FFT 和实时数据，独立给一人；ANIM 涉及帧动画和位图，独立给一人 |
| **页面数量** | SETTING + BRIGHT 逻辑简单（各约 50 行），合并给一人 |
| **负责人全栈覆盖** | 负责人完成核心模块后，组员只需关注自己的 `.cpp`，不需要碰架构 |

### 并行开发约束

- 6 个页面之间**零耦合**：每个页面的 `.cpp` 只 include 公共头文件，不互相依赖
- 核心模块头文件（`store.h`、`state_machine.h`、`page_base.h`）**冻结**，组员只读不写
- 每人只需在 `src/pages/xxx_page.cpp` 一个文件中写代码
- 位图/乐谱数据文件**核心组维护**，页面组通过 `img.h` / `songs.h` 只读引用
